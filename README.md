Este repositorio contiene los Acceptance Tests desarrollados para el proyecto **SmartBus Tech**, elaborado para el curso de Ingeniería de Requisitos.

Las pruebas de aceptación fueron implementadas utilizando **Behavior-Driven Development (BDD)** y redactadas en lenguaje **Gherkin**, siguiendo la estructura:

- Feature
- Background
- Scenario
- Scenario

Cada archivo `.feature` corresponde a una User Story definida durante el Capitulo 3: Requirements Specification.

## 🗂️Organización de las carpetas

- **EP002_ControlRecaudacion:** pruebas de aceptación correspondientes al proceso de recaudación diaria y liquidación de caja.
- **EP005_InteligenciaNegocio:** pruebas de aceptación relacionadas con dashboards, KPIs y reportes estratégicos para la toma de decisiones.
- **RequisitosNoFuncionales:** pruebas de aceptación relacionadas con requisitos no funcionales de autenticación, autorización y seguridad.
